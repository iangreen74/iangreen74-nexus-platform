# ARIA Intelligence — How ARIA Thinks

**Status:** Operational as of April 23, 2026. All substrate phases deployed.
**Audience:** Engineers working on ARIA, investors understanding the moat.

---

## What ARIA Is

ARIA is a technical co-founder, not an assistant. The distinction matters:

- An assistant waits for instructions. ARIA speaks first.
- An assistant forgets between sessions. ARIA remembers everything.
- An assistant knows the product. ARIA knows the *founder*.

The intelligence system exists to make ARIA intimate — to know not just what the founder is building, but how they think, what they worry about, what they've tried before, and what they're carrying.

---

## The Prompt Assembly Pipeline

Every ARIA response flows through `assemble_aria_prompt()` in `nexus/aria/prompt_assembly.py`. The function composes a system prompt from six data sources:

```
1. Persona         — who ARIA is (from persona.md, never trimmed)
2. Founder context — who you're talking to (from Neptune UserContext)
3. Rolling memory   — what you remember (daily/weekly/monthly summaries)
4. Tone context    — recent emotional weather (from tone_markers)
5. Active ontology — what you know about the product (Features/Decisions/Hypotheses)
6. Socratic        — what you might want to think about (from Mechanism 3)
7. History         — recent conversation turns
```

Each source is read independently. Each has graceful fallback to empty/default on error. The prompt assembly never fails — it always produces a valid system prompt.

## Token Budget Architecture

Total budget: **10,000 tokens** (~40,000 characters at 4 chars/token).

| Section | Budget | Priority | Trim behavior |
|---------|--------|----------|---------------|
| Persona | ~2,000 | 0 | **Never trimmed** |
| Founder context | ~200 | 10/20 | Trimmed late |
| Ontology | ~500 | 30 | Trimmed mid |
| Rolling memory | ~500 | 40 | Trimmed mid |
| Tone | ~100 | 50 | Trimmed early |
| Socratic | ~200 | 60 | Trimmed early |
| History | ~6,500 | 100 | **Trimmed first** |

When total context exceeds budget, sections are halved starting from highest priority number. History is always trimmed first. Persona is never trimmed — ARIA's character is non-negotiable.

## Memory Compression Chain

Raw conversation history grows unbounded. The compression chain keeps it bounded:

```
Raw turns (unlimited) → nightly → Daily digest (~500 tokens)
                                      ↓
7 daily digests → Monday 2am → Weekly rollup (~300 tokens)
                                      ↓
~4 weekly rollups → 1st of month → Monthly arc (~200 tokens)
```

Each level compresses the level below, never re-reading raw data. A 6-month-old monthly arc captures the narrative shape of that month in 200 tokens: "April was the month you stopped trying to do it alone."

Generated by Haiku (us.anthropic.claude-haiku-4-5-20251001-v1:0). If Haiku fails, structural fallback: "N conversations, M features shipped, dominant tone: Y." Never empty.

Storage: `rolling_summaries` Postgres table, UPSERT on `(tenant_id, horizon, for_date)`.

## Tone Classifier (Phase 5)

Every founder conversation turn is classified for emotional texture:

| Dimension | Values |
|-----------|--------|
| tone | relaxed, curious, excited, anxious, frustrated, decisive, uncertain, confident, defeated, energized |
| urgency | low, medium, high |
| seeking | advice, validation, execution, company, clarity, nothing_specific |
| mentions | named entities (people, products, competitors) |

Classified by Haiku per-turn. Written to `tone_markers` Postgres table. Read by prompt assembly as "Recent emotional weather."

Purpose: ARIA responds to the emotional arc, not just the literal words. A founder who's been `anxious, anxious, anxious` for three turns needs a different response than one who's been `excited, decisive, energized`.

## Socratic Primitive (Mechanism 3)

ARIA doesn't just respond — it notices. The Socratic scheduler runs hourly, scanning each tenant's ontology for patterns that warrant proactive questions:

| Rule | Triggers when | Question shape |
|------|--------------|----------------|
| stale_hypothesis | Hypothesis >7d, no linked Decision | "You hypothesized X a week ago. Want to test it?" |
| dormant_decision | Decision >30d untouched | "You decided X a month ago. Still the right call?" |
| built_not_deployed | Feature built, no deploy in 48h | "X is built but not deployed. What's blocking it?" |
| deploy_failure_streak | 3+ failures in 24h | "Three deploys failed today. Want to look at what's common?" |

Rules are deterministic (no Haiku) — they scan the ontology graph via openCypher queries. Prompts are persisted to `socratic_prompts` Postgres table with UNIQUE constraint preventing duplicates.

The prompt assembly reads pending Socratic prompts and injects them as "What you might want to think about" — a gentle nudge, not an alert.

## The Capture Loop

The complete end-to-end loop:

```
Founder speaks (conversation turn)
  → EventBridge: conversation_turn event
  → Lambda: classifier.extract() + tone.classify_tone()
  → Proposals written to classifier_proposals
  → Tone markers written to tone_markers
  → UI renders proposal cards (Phase 8b)
  → Founder accepts/rejects/edits
  → Accept: POST /api/ontology/propose_object
  → Neptune MERGE + Postgres version write
  → Next ARIA prompt grounds in the new ontology object
  → ARIA's response reflects what the founder just decided
```

This is the compounding loop. Every founder interaction makes ARIA smarter about that founder.

## Known Gaps

1. **No Bug type** in ontology v0. ARIA can't ground responses in bug history. Important for the "calm under pressure" persona commitment.
2. **UserContext is product-focused** (product_name, product_vision, target_users), not founder-focused (no founder_name, stage, prior_attempts). The intimacy thesis requires schema evolution.
3. **Eval corpus substrate-only.** S3 bucket exists, write policy exists, but no ActionEvents are actually flowing yet. Loop 2 (weekly classifier improvement) blocked until events flow.

## The Intimacy Thesis

ARIA knows you after months, not minutes. The combination of:
- Ontology (what you've built, decided, hypothesized)
- Tone markers (how you've felt)
- Rolling summaries (what happened this week, this month)
- Socratic prompts (what you should be thinking about)

...produces a co-founder who remembers your journey. Not a chatbot that starts fresh every session.

This is the moat. Competitors can match the AI model. They can't match six months of accumulated founder intelligence.
